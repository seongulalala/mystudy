# 업무 흐름(제품 분석)

## 데이터 분석 관점에서 회사에서 해야하는 일
- 현재 상황 파악
- 특정 기능이 출시된 후, 기능의 성과를 확인
- 그래서 우리는 무엇을 해야할까? > Action을 도출

## 제품 분석의 큰 흐름
서비스, 비즈니스 파악 > 대표적인 지표 확인(이벤트 횟수, 퍼널, 리텐션) > 문제 확인 + 가설 > Action논의 > 기능 개발/배포 > 기능의 핵심 지표들

## BigQuery 환경 구성 요소
1. 프로젝트
- 하나의 건물
- 하나의 프로젝트에 여러 데이터셋 존재 가능
- 주로 2개(실전용, 개발용)

2. 데이터셋
- 프로젝트에 있는 창고
- 각 창고 공간에 데이터를 저장

3. 테이블
- 창고에 있는 선반
- 행과 열 구성
