# 정리
#### 리텐션 분석 핵심
> 리텐션과 기간을 정의하고 / 리텐션 커브를 그리고 / 추가 분석을 진행한다

#### 정의의 핵심
> 1  
> 우리 제품/ 서비스는 고객의 어떤 문제를 해결하려고 하는지를 고려하여  
> 핵심 이벤트(제품/서비스의 핵심)을 리텐션으로 잡음

> 2  
> 앱 접속을 리텐션으로 잡음


#### 기간의 핵심
> 사용자의 재사용 분포를 **히스토그램**으로 확인해 결정 or 기타 인사이트

---
# 리텐션 분석의 큰 흐름
1. 서비스 비즈니스 파악(서비스의 목표,문제,유저 흐름 등)

2. 문제 정의(핵심 문제, 목표 정의, 핵심 이벤트 정의)

3. 사용 주기 파악 - 고객은 문제를 어떤 빈도로 겪을까 파악 > 리텐션 기간을 설정(일별/주별/월별)

4. 리텐션 커브 분석 - PMF도달 여부 확인 - 평평해지는 시점 확인

5. 추가 분석 - 코호트, 유저 행동 분석(인구통계,기기,국가). 유저 쪼개기(신규/기존)

6. 팀 공유

7. 기능 개발 및 배포

# 리텐션에 대해
- 리텐션 그래프를 그려보고 판단한다
- 여러 가지 케이스를 나누어 그려본다

## 리텐션 핵심
#### 리텐션에 따른 정의가 필요함
    - ex) 앱 접속 , 특정 이벤트

#### 주기에 따라 관점이 달라짐
    - ex) 접속 주기가 서비스별로 다름

## 제품의 핵심
#### 우리 제품/서비스는 고객의 어떤 문제를 해결하려고 하는가?
> 이 내용을 고려해야 리텐션이 쉬움

> 제품/서비스의 핵심을 생각해야 함

### 리텐션 정의
1. 핵심 이벤트(제품/서비스의 핵심)
2. 앱 접속


## 리텐션의 종류
1. Classic리텐션 = (N day리텐션)
    -  사용자가 최초로 "참여"한 Day 0 이후, 각 Day N별로 한 번 더 "참여"했는지 계산

2. Range 리텐션 = (Bracket 리텐션)
    - 특정 Range 내에서 더 "참여"했는지 계산
    - 특정 범위 안에 한번아라도 참여하면 "참여"

3. Rolling 리텐션 =(Unbounded 리텐션)
    - Day N 이후에 한 번이라도 "참여"한 경우를 모두 참여했다고 간주


## 종류별 주요 사용 상황
- Classic   
    = 사용자가 주로 매일 or 정기적으로 사용하는 서비스인 경우   
    = 마케팅 캠페인을 한 후, 반응을 확인하는 경우  

- Rolling   
    = 사용자 사용 주기가 아주 긴 경우

- Range   
    = 주로 실무에서 사용



## 리텐션 커브 그래프의 핵심
> 핵심은 리텐션 커브를 위로 올리기  
> 평평한 리텐션이 언제 생기는가를 확인
