# 첫번째 배운점
결론이 너무 공격적, 섣부르고 분석가로서 추가적인 분석 (ex 앱 알림 on/off에 대한 ab테스트 등)을 액션 플랜으로 제안했어야했어야 함  

```
# 네가 한 흐름

리텐션 낮음
→ 앱 매력 부족
→ 제휴 채널 비효율

# 분석가가 기대되는 흐름

리텐션 낮음
→ 구조적 원인 후보 도출
→ 검증 가능한 추가 분석 / 실험 제안
→ 그 결과로 판단

즉,
판결을 먼저 나중에 찾은 구조라서 공격적으로 보였음
```
좋았을 액션 플랜
```
알림 ON vs OFF 사용자 리텐션 비교

D2 소액 보상 A/B 테스트
```

# 세번째 배운점

```
# 인사이트 표현 공식 

[데이터로 관찰된 사실] +
→ [서비스 구조/메시지와의 불일치]

# KPI 표현 공식

[핵심 행동의 도달률 or 유지율] = 주로 비율
(시간·횟수는 보조 지표) = 주로 절대값
```

예시
```
# 인사이트

D0에 식단 사진을 2회 이상 기록한 사용자의 D14 리텐션이 46%로,
1회만 기록한 사용자(18%) 대비 크게 높게 나타난다.
> 
이는 “귀찮지 않게 식단 관리”라는 핵심 가치와 달리,
실제로는 초기 반복 행동을 수행해야 서비스 가치를 체감할 수 있는 구조임을 시사한다.

# KPI
Primary KPI:
D0 식단 기록 도달률 (meal_log ≥ 1)

Secondary KPI:
첫 식단 기록까지 걸리는 시간 (median)
```
