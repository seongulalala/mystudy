# 데이터 분석을 위한 일반적인 프로세스
> 기획 & 데이터 수집 기획(비즈니스 분석) > 데이터 로그 설계(이벤트 설계 등) > 분석툴 설치 및 셋팅(GA4) > 데이터 수집 > 데이터 분석


# 기본 마인드
> 데이터를 보고 할 때 기억해야 하는 사항
1. 기간
2. 지표
3. 판단 기준(비교)

> 분석의 관점
- 요약 정보를 보고 > 판단하고 > 요약 정보를 펼친다(쪼갠다)

# GA4 기본 용어
- 세션 : 사용자가 사이트에 들어와서 활동하다 떠날 때까지의 한 덩어리(= 한 번의 방문)

- 이벤트 : 사용자의 행동 EX) 페이지 뷰, 클릭, 스크롤

- 전환 : 이벤트 중 중요한 것 EX) 구매, 구독

- 세션 VS 사용자 VS 이벤트

    |개념| 설명|
    |-----|----------|
    |세션 | 한 번의 방문|
    |이벤트 | 방문 중 발생한 행동|
    |주요 이벤트| 전환 (구매, 구독 등)
    |사용자 | 사람|

> 1개의 세션에 여러 이벤트가 있을 수 있다  
> 여러 이벤트가 있더라도 전환은 하나일 수 있다

- 참여 세션 : 아래 3가지 조건 중 1개라도 만족한 세션
    1. 10초 이상 지속
    2. 2페이지 이상 조회
    3. 전환이 일어남(주요 이벤트)

## 세션에 대해 조금 더
- 세션은 한 번의 방문이다

- 이를 나누는 기준

    1. session_start 발생
        - 사용자가 처음 들어왔을 때 무조선 새 세션 발생
    
    2. 30분 비활동 > 세션 종료
        - 이후 다시 행동 > 새 세션
        - 이 30분은 시간 설정 가능

    3. 캠페인 정보가 바뀌면 새 세션
        - 같은 사용자이지만 utm_source / utm_medium으로 변경되었다면 새 세션

# GA4 보고서 정리
## 비즈니스 흐름 vs 사용자
- 비즈니스 흐름 기준
    - 유입 >> 행동 >> 성과 >> 재방문

- 사용자 = 사용자에 관해
    - 사용자 속성, 기술 

## 비즈니스 흐름 구조
- 유입 = 획득
    - 어디서 들어왔는가?
    - 마캐팅 성과, 채널 비교에 사용

- 참여도 = 행동
    - 들어온 사람들이 의미 있게 썻는가?
    - 품질 상태 점검에 사용(ex 사용성 - 어디서 고객이 나가지?)

- 수익 창출
    - 그래서 돈을 벌었는가?
    - 매출, 구매 성과 판단에 사용

- 유지
    - 사람들이 다시 돌아오는가?
    - 재방문, 리텐션 확인에 사용

## User 구조
- 사용자 속성
    -  개요 
        - 누가 쓰고 있나?
        - 사용자 분포 요약
    
    - 인구통계 세부정보
        - 연령 성별 지역 등이 어떻게 되는가?
        - 타겟 검증, 광고 타겟 검정에 사용

    - Audiences
        - 어떤 사용자 집단이 필요한가?
        - 잠개 고객 파악에 사용

- 기술
    - 개요
        - 어떤 환경에서 쓰나?
        - 플랫폼, 디바이스 , 운영 체제 등
 
    -  기술 세부정보
        - 어떤 환경에서 쓰나?
        - 플랫폼, 디바이스 , 운영 체제 등
