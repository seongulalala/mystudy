## 기여모델 사용 상황
- 고객이 구매 전 어떤 채널들을 거쳐 웹사이트에 도달했는가
- 전환을 만들기 위해 마케팅 비용이 어떻게 사용되었는가
- 각 채널의 **수익성(ROAS, 전환 기여도)**은 어느 정도인가
- 전환에 실제로 기여한 채널은 무엇인가
- 앞으로 예산을 유지·확대할 채널은 어디인가


#### 기여 모델은 전환에 기여한 채널을 공정하게 평가해 마케팅 예산 배분을 결정하기 위해 사용한다.

## 기여모델
**네이버 키워드 광고**를 보고 **네이버 블로그**를 방문 한 뒤
**구글 자연 검색**으로 웹사이트에 들어와 구매를 했을 때,  

구매 전환은 어느 채널로 잡혔을까?

> 당연히 구글 자연검색


- 마지막 상호 작용  = 구글 자연 검색 Organic search
- 마지막 간접 클릭 = 네이버 블로그 referral
- 지원 전환 성과 = 네이버 키워드 광고

## 기여모델 이해
GA4는 기본적으로 데이터 기반 기여 모델을 채택했다.(광고란에서)

데이터 기반 모델은 머신러닝 알고리즘을 사용하여 개별 전환 경로 데이터를 분석하고, 각 채널이 전환에 기여한 정도를 계산해 기여도를 할

**Direct 채널은 전환 기여에서 우선순위가 낮아** **마지막 비-Direct 채널**로 대체될 수 있음


ex) direct > referral = referral 
ex) referral > direct = referral  
ex) direct > direct = direct  
ex) Organic Serarch > direct > direct = Organic Serarch   
ex) Paid vidio > direct = Paid vidio  


## 그럼 기여모델을 보려면 어디로 가야할까
> 광고 > 주요 이벤트 > 주요 이벤트 기여 분석 모델


## 햇갈리는 3가지 캐이스

| 구분        | 트래픽 획득 | 광고 기여 분석 | 탐색    |
| --------- | ------ | -------- | ----- |
| 기준 시점     | 세션 시작  | 전환 발생    | 이벤트   |
| 목적        | 유입 분석  | 기여 분석    | 구조 분석 |
| Direct 처리 | 그대로 표시 | 대체됨      | 원본값   |
| 기여 모델     | ❌      | ✅        | ❌     |
| 실무 용도     | 유입 현황  | 성과 판단    | 원인 분석 |


### ★실무에서 이렇게 쓰면 됨★

#### “유입이 뭐였지?” → 트래픽 획득

#### “광고 성과 누구 공이야?” → 기여 분석

#### “왜 이런 결과가 나왔지?” → 탐색

## 기여모델 이해
- 데이터 기반 기여 모델은 광고란에서만 사용된다
- 광고 기여 분석에서는 전환을 나눠 계산해서
숫자가 소수점으로 나오는 게 정상이다.

- 사용자는 보통 여러 채널을 거쳐 전환하므로, 마지막 채널만 보면 앞단에서 영향을 준 채널의 역할이 사라진다. 그래서 전환을 나눠 각 채널의 실제 기여를 더 정확히 본다.
