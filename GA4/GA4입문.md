# 설치
## GA4 계정 만들기
1. 회사 웹사이트의 모든 페이지에 나의 GA4 스크립트를 넣어야 함
2. head 테그 안에 넣어야 함

## 태그 관리자 만들기
1. **컨테이너** 이름은 회사 웹사이트 주소
2. 회사 웹사이트의 모든 페이지에 나의 데그 관리자 스크립트를 넣어야 함
3. head 테그 안 + body 테그에 넣어야 함

# 기본 개념
- GA4
    - 데이터 분석 조회 도구
    - 태그를 설치하지 않으면 데이터 없음

- GTM
    - 테그, 트리거 관리 도구
    - 태그, 트리거를 만들어 웹사이트와 GA4를 연결하는 곳


## 태그 관리자(GTM)
- 태그 관리자와 웹사이트를 연결하면 태그 관리자에서 수정을 하면 웹사이트에 적용됨

- 웹사이트 코드를 직접 수정하지 않고 버전 관리를 통해 안전하게 웹사이트를 운영할 수 있음

## 구성
### 작업 공간
- 태그  : 데이터를 보낼 곳
- 트리거(방아쇠) : 시점(~할 때)
- 변수 : (데이터를 담을 수 있는) 그릇

- ex) 트리거 : 모든 페이지에 사용자가 오면
- ex) 태그 : GA4로 보내줘

#### 트리거 만들기
- 기본적인 트리거는 트리거 유형에 있다
- 페이지뷰 / 클릭 / 동영상 시청 등

#### 태그 만들기
- 만든 트리거 중 하나 설정 Ex) 모든 페이지뷰 트리거 선택

### 버전
- 사이트에 이상이 있을 때, 버전을 롤백할 수 있다
- 미리보기 공유 > 다른 사람에게 내가 만든 버전에 대한 작업된 웹사이트를 볼 수 있다

### 관리자
- 해당 웹사이트를 관리할 사람에게 권한을 줄 수 있음(이메일 주소)
- 권한에 대한 레벨을 관리할 수 있음


## GA4
- 권한 주기 : 관리 > 계정 액세스 관리, 속성 액세스 관리

### 계정 액세스
- 회사 전체 GA4에 접근 가능
- 회사 GA 안에 3가지 웹사이트가 있다면(쇼핑몰, 공식 사이트 , 블로그) 이 모두에 대한 권한이 주어짐

### 속성 액세스
- 웹사이트에 대한 권한
- 특정 사이트(쇼핑몰)에 대한 권한만 주어짐
