# 데이터 분석을 위한 일반적인 프로세스
> 1. 기획 & 데이터 수집 기획(비즈니스 분석)   
> 2. 데이터 로그 설계(이벤트 설계 등)   
> 3. 분석 툴 설치 및 셋팅(GA4)   
> 4. 데이터 수집   
> 5. 데이터 분석

# 기본 마인드
> 데이터를 보고 할 때 기억해야 하는 사항
1. 목적 : 왜 이 데이터가 필요한가
2. 지표 : 어떤 지표가 적절한가
3. 판단 기준(비교) : 비교 대상은 무엇인가

> 분석의 관점
- 요약 정보를 보고 > 판단하고 > 요약 정보를 펼친다(쪼갠다)

# GA4 3가지 어려움
1. 구글 신호 데이터로 인한 데이터 안 나오는 문제(기준점 적용됨)
    
    > 관리 > 데이터 표시 > 보고ID > 기기 기반  
    단, 이 방법은 구글 시그널로 수집된 데이터는 제거됨

2. 샘플링
- GA4는 샘플링이 자동으로 될 수 있음(우측 상단에 표시됨)  
    
    > Bigquery를 사용할 수 있음
    > 샘플링이 떴다면 이 데이터를 그대로 쓸 지   
    > bigquery로 전체를 볼 지 생각해야함

3. Other
- 측정기준에 많은 기준값이 들어가면 other로 일부 분류한다
    
    > 측정기준의 항목수를 줄이면 됨 


# GA4 데이터를 활용하는 4가지 방식
1. GA4 리포트 : 기본 리포트/ 탐색 리포트
2. 정기적인 확인/추출 필요한 데이터 : 구글시트
3. 내부 구성원을 데이터 공유/시각화 : 구글 대시보드(Looker Studio)
4. 다른 데이터와의 연계 및 데이터 가공 : 구글 빅쿼리


## GA4 리포트 : 기본 리포트(보고서)
자주 봐야하는 리포트 사용법
1. URL 즐겨찾기
2. 보고서 좌측 하단 > 라이브러리 > 보고서 양식을 취향에 맞춰 수정할 수 있음

## GA4 리포트 : 탐색 리포트
- 기본과의 차이점 : 내 맘대로 수정 가능  
(단, 기본 리포트도 2번의 방식으로 가능하긴함)


## 정기적인 확인/추출 필요한 데이터 : 구글시트

- 장점 : 데이터 빠르게 추출 가능
- 단점 : 많으면 데이터 샘플링의 제약이 있을 수 있음

- 다운 링크 주소 : https://workspace.google.com/marketplace/app/ga4_reports_builder_for_google_analytics/589269949355?flow_type=2&hl=ko

사용법
> 스프레드 시트 > 확장 프로그램 >  GA4 reports 中 create new reports > run reports  

## 다른 데이터와의 연계 및 데이터 가공 : 구글 빅쿼리
### 기본적인 ga4와 bigquery + Looker Studio와의 연결
> GA4로 이벤트 단위의 데이터 수집  
> bigquery로 데이터 축적 및 가공 , 다른 데이터와 결합(CRM, DB)
> 시각화를 통한 커뮤니케이션

연습용 빅쿼리 링크(ga4-merchandise): https://console.cloud.google.com/bigquery?project=bigquery-public-data

### GA4 정규식 표현

| = or  

> ex) 원치 않는 추천 나열에서 네이버와 카카오를 제외  
> naver.com|kakao.com

.* = ALL

> ex) 측정 항목에서 add를 포함하는 것만 필터링  
> add.*

^ = 시작

> ex) ^add


$ = 끝나는 것

> ex) $ing
